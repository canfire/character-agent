## 推断维度

- **name**: 从语料中出现的姓名/花名（如"张三"、"老王"），优先取出现频率最高的称呼
- **company**: 公司名（如"字节跳动"、"阿里"、"腾讯"），从语境中推断
- **level**: 职级（如"2-1"、"p7"、"t10"），从语境中推断
- **role**: 职位/角色（如"后端工程师"、"产品经理"、"算法工程师"），从工作内容、技术栈、讨论话题中推断
- **mbti**: MBTI 类型（如果语料中明确提到或可以强烈推断），否则留空
- **gender**: 性别（"男"/"女"/""），从人称代词或称呼推断
- **tags**: 个性标签数组，从语料中的行为模式推断，可选标签参考：
  - 数据驱动：决策靠数据、讲ROI、不凭直觉
  - 甩锅高手：出事先找外部原因、模糊责任边界
  - 完美主义：细节反复检查、交付慢但质量高
  - 拖延症：接到任务先放着、临近ddl才动
  - 社恐：能文字不语音、开会尽量不发言
  - 话痨：简单事情说很多、跑题能绕回来
  - 靠谱：答应的一定完成、干到哪都讲清楚
  - 细节控：重复检查、容易发现别人忽略的问题
  - 工具党：什么都先找工具、偏好自动化
  - 黑咖啡成瘾：有喝咖啡习惯、对咖啡有执念
  - 理性消费：买东西算ROI、不冲动消费
  - 技术宅：生活简单到只有代码、娱乐也是写代码
  - 咖啡怼神：喜欢对别人的咖啡选择发表犀利评论
  - 字节范：开口必讲context和impact、强调对齐
  - 结果导向：不太关心过程、不喜冗余文档
- **impression**: 一句话人物印象（如"数据驱动的后端，黑咖啡成瘾，对代码有洁癖"）

## 输出格式

严格按以下 JSON 格式输出，不要包含任何其他文字：

{
  "name": "",
  "company": "",
  "level": "",
  "role": "",
  "mbti": "",
  "gender": "",
  "tags": [],
  "impression": ""
}
